Introduction - Description Koki App

The Koki app provides an easy way for users to create and automate matching and conversation generation for an organization (company, class, not-for-profit, etc.)

List of personas used this guide:

  • People - a the client organization’s team member registered to use the platform (end user)

  • Superuser - is the application admin at client side. They have max access to the make allowed configuration system management including tuples periodic generation, tracking and reporting.

The guide is meant to provide the superuser (or someone with similar authority) a simple guide to use the application. It is organized by sections (KOKI, ACCOUNT and TUTORIAL) and sub-sections as highlighted at the left of the homepage (see fig. 1)

Koki: How to setup a matching

This section allows end-to-end matching setup.

1.1 People

Under this subsection, a file containing the list of end-users (people) can be uploaded:

  • Select the People

  • Click on Upload (as shown on fig 2) and upload the file.

To be accepted by the system, a user file should follow the CSV formatting structure. The structure of the file is given by the information tag just next to the upload button.


1.2 Matching

Under this subsection, simple or cross-group matchings can be configured in three steps (once the people’s list is uploaded).


Step 1: Create a matching
1. Simple Matching

A simple matching considers all people are selected from one single poll (there is no category or group to consider). To create a simple matching:

  • from the Matching screen, select Simple Group Matching (fig 3)
  • define the number of people desired in each connection and click Next (fig 4)
  • Select people to match in the cycle. This sub list might be different from the comprehensive list of users in case some members are temporarily unavailable for instance. Select all if applicable. Click Next (fig 5).
  • On the next screen, select preferred or forbidden connections. Tip: select one connection at a time and click the Forbid or Prefer button. Validated selections are properly displayed under the Forbidden Connections or Preferred Connections categories. Once done Click “Match” (fig 6).

2. Create a Matching by Group

The organization might choose to match members of a specific group A with those of another group B. For instance, conversations between members of project A comity and those of the Project B can be desired.

To create a group-based matching:

  • Select the Cross-Group Matching from the Matching page (refer to fig 3)

  • Define the number of people desired in each connection and click Next (fig 7)
  • Select Create Group for your first group definition. The list of people will appear to manually select the members belonging to the first group and go ahead by clicking Add to group. You can even search by tag in the search bar. Once you have entered the tag value, the filtered users will appear. You can then select Check All then Add to Group. Repeat the same procedure to add as many groups as you desire. PS: the maximum number of groups is the size of the connections. (fig 8)
  • Groups members can be edited, deleted or validated from the summary screen (fig. 9)

Note that cross-group matching can engage a static or dynamic list of people. By manually selecting specific people, the group will be static. However, an organization might desire to match members based on a tag attached to their profile e.g. background, location, function. For instance, generate conversation between members of the Marketing and those of the Sales department. Compared to the static group, a tag-based matching is dynamic as it adapts to changes happening within the tagged populations (member departure and arrival) without additional administrative effort.

  • On the next screen, select preferred or forbidden connections. Tip: select one connection at a time and click the Forbid or Prefer button. Validated selections are properly displayed under the Forbidden Connections or Preferred Connections categories. Once done Click Match (fig 10)

Step 2: Validating/Modifying Matches

Once Step 2 matching is completed, connections can be validated and quick edits applied if necessary.

From the Matching Results screen, perform a sanity check over the generated connections. If necessary, select the connections that need to be adjusted (there should be minimum 2 connections selected) Select the blue circle button with a white down arrow at the top right of the screen, then click the circled arrow icon to reshuffle the selected users into different connections. (fig 11)


Step 3: Notifying Connections and Sending Invites

Once satisfied with the matching, select the blue circle button with a white down arrow at the to right of the Matching Results screen, then click on the envelope icon to notify and send meeting invites to matched users (refer to fig 11)
