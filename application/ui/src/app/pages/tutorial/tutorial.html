Introduction - Description Koki App

The Koki app provides an easy way for users to create and automate matching and conversation generation for an organization (company, class, not-for-profit, etc.)

List of personas used this guide:

  • End user - a client's team member registered to use the platform
  • Superuser - is the application admin at client side. They have max access to the make allowed configuration system management including tuples periodic generation, tracking and reporting. A superuser is not necessarily a user
  • Lead Superuser - the who the contact with the application provider. They approve, validate, and change Client requests.
  • Server Administrator - an agent at the app provider side who has reasonable access the platform
  • Super Administrator - an administrator with full access and grant or restrict access

The guide is meant to provide superuser (or someone with similar authority) simple instructions to use the application. It is organized by section highlighted at the left of the welcome page (see fig. 1)

Koki: How to setup a matching

An end-to- matching setup can be the following steps:

Step 1: Uploading a user file

To upload a user list, select the Users section at the left of the home page, on Upload at the top right (fig 2) and upload the file. To be accepted by the system, a user file should follow the CSV formatting structure . The structure of the file is given by the information tag just next to the upload button of the file is given by the tag just next to the upload button [ need shot for this]

Step 2: Generate a matching

a. Create a matching

To create a matching:

  • select the New Match section at the left of the home page,
  • define the size of that will connect click Next (fig 3)
  • select the users to match in the cycle. This sub list might from the comprehensive list of users in case some members temporarily unavailable for instance. Select all if applicable. Click Next (fig 4).
  • the next screen, select preferred or forbidden tuples. Tip: select tuple at a time and click Forbid or Prefer button. Validated the selections properly displayed under the Forbidden Connections or Preferred Connections categories the right . Once done Click Match (fig 5)

b. Create a Group

The organization might choose match members of a specific group A with those of another group B. instance, conversations between members of project A comity and those of the Project B can be desired. Note that, the

create a group-based matching:

  • the New Group Match section at the left of home page,
  • define the size of tuple that will connect by clicking oMatching Size
  • each group, members to be matched by on grey arrow beside Users Group X (fig)
  • the Users Group X selection window will then appear to allow group users selection. Click Add once the selection is completed (fig ). Repeat at least as many as the tuple size.
  • tuples can also selected from screen (like group members selection) (fig 5)

Note that for a group-based matching, group members are selected, and list is static. For a more group -based selection, a tag-based matching might be more appropriate.

c. Matching by Tag

An organization might desire to match members based on a tag attached to their profile e.g. background, location, function. For instance, generate conversation between members of the Marketing and those of the department. Compared to group-based, a tag-based matching is it adapts to changes happening within the tagged populations (member and arrival) additional administrative effort.

To create a tag-based matching:

  • select the New Tag Match section at the left of the home page,
  • define the size of tuple that will connect by clicking on Matching Size
  • for each group, define or select members be matched by clicking on grey arrow beside Users Group X (fig)
  • the Users Group X selection window then appear to allow group users selection. Click Add once the selection is completed (fig ). Repeat at least as many groups as the size.
  • forbidden tuples also be selected from screen (like selection) (fig 6)

Step 3: Validating/Modifying Matches

2 matching is completed, tuples can be validated, and quick edits can necessary.

  • From the Matching Results screen, perform a sanity over the tuples generated
  • If necessary, the tuples that need to be adjusted (there should be minimum 2 tuples selected)
  • Select the circle button with a white at top right of screen, then click the circled arrow icon to reshuffle the users into different tuples. (fig)
